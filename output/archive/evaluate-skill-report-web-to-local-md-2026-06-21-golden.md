## 改进建议

### 🔴 优先级 1：脚本核心功能修复

1. **建议在脚本 `web_to_local_md.py` 中修复图片下载逻辑**：当前脚本对 Anthropic CDN（`/_next/image?url=` URL编码）、byteimg CDN（`.awebp` 签名 URL）和 WordPress 嵌套图片（`<table><td><figure><img>`）三种图片提取场景均失败。需要：
   - 解码 URL 编码的图片路径（`%3A%2F%2F` → 实际 URL）再下载
   - 支持带签名参数的 CDN URL 图片下载（byteimg）
   - 支持 WordPress `<table><td><figure><img>` 嵌套结构中图片的提取
   - 对无法自动下载的图片，至少将 CDN URL 替换为本地占位路径并标注 `[图片暂缺]`

2. **建议在脚本中添加 `<small>` 日期标签生成逻辑**：SKILL.md 常见错误 #6 明确要求 "Add `<small style="color:gray">YYYY-MM-DD</small>` below the H1 title"。当前脚本 **7/7 用例全部缺失日期标签**，这是最普遍的合规违规。需在 HTML 提取后自动检测发布日期（常见位置：`<time>` 标签、meta 标签、文章头部日期文本），并按 SKILL.md 格式插入。

3. **建议在脚本中添加 H1 标题提取逻辑**：4/7 用例缺失文章 H1 标题。需从 HTML 的 `<h1>` 标签或页面 `<title>` 中提取标题，确保输出文件以 `# 标题` 开头。

### 🟡 优先级 2：噪声剥离增强

4. **建议在脚本中增强噪声剥离规则**：当前脚本未剥离以下类型的网站噪声：
   - 作者署名链接（如 `[CodeSheep](/user/...)`）
   - 阅读量/阅读时间指标（如 `18,755` / `阅读6分钟`）
   - 作者/分类标签（如 `业务研发平台` / `算法大模型`）
   
   建议在 SKILL.md 的 "Strip noise" 规则中补充：`剥离作者署名、阅读量、阅读时间、分类标签等非文章核心内容的元数据`，并在脚本中实现对应的 CSS selector/正则剥离逻辑。

5. **建议在 SKILL.md 常见错误表增加规则**：

   > | Mistake | What happens | Fix |
   > |---------|-------------|------|
   > | 未剥离文章元数据（作者链接、阅读量、阅读时间） | 输出包含非核心内容噪声，影响阅读体验 | 使用 CSS selector 剔除 `.author`、`.view-count`、`.read-time` 等元数据区块 |

### 🟢 优先级 3：输出质量提升
