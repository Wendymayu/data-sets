## 共性问题（出现在 ≥3 个用例中）

### 1. 图片 alt 文本为哈希值而非描述性（6/7 用例）

输出文件的图片 alt 文本普遍为哈希字符串或维度字符串（如 "1c5fff78273feaf4892b46ad3fb757956195300"、"P873364"、"4584x2580"），而参考文件中的 alt 文本均为描述性中文（如"重构时间线与执行路径"、"阿里云端到端链路追踪架构图"）。fill_image_alt_text() 的规则（strip ext, replace -/_ with spaces）只能从文件名推导，无法产生语义描述。

受影响用例：case-1, 2, 3, 4, 5, 7（case-6 因来自原始 .md 文件，alt 文本较好）

### 2. 发布日期缺失或位置错误（5/7 用例）

SKILL.md 规定 `<small style="color:gray">YYYY-MM-DD</small>` 应放在 H1 标题下方。实际输出中：
- case-1, 2: 缺少日期标签
- case-3, 5, 7: 日期在 H1 上方而非下方

受影响用例：case-1, 2, 3, 5, 7

### 3. 图片引用与文件脱钩（3/7 用例）

- case-1: 4 张不同图片映射到同一简化引用名 `4584x2580.png`
- case-5: 脚本崩溃导致所有图片仍为 CDN URL
- case-7: 26 张图片文件下载但零引用注入 markdown

受影响用例：case-1, 5, 7

### 4. 噪声内容未清理（2/7 用例，未达阈值）

- case-4: AICon 会议推广段落保留
- case-5: 反斜杠转义粗体残留

虽未达 3 个用例阈值，但仍是可改进点。

## 改进建议

### 1. 增强 fill_image_alt_text 规则，支持语义化 alt 文本推导

建议在 SKILL.md 的常见错误 "Empty image alt text" 规则中补充：当 HTML 原始图片存在 `<figcaption>`、`<title>`、`alt` 属性或周围有描述性文本时，应优先采用这些文字作为 alt，而非仅从文件名推导。文件名推导仅作为 fallback（当前行为）。涉及步骤 3 "Download images + fix paths" 中的 fill_image_alt_text() 函数。

### 2. 明确发布日期位置规范：必须紧接 H1 标题下方

建议在 SKILL.md 的常见错误 "Missing publication date" 部分增加明确规则：`<small style="color:gray">YYYY-MM-DD</small>` 必须放在 H1 标题紧接下一行，不得放在标题上方或与封面图之间。涉及步骤 3 中的 extract_page_metadata() 函数。

### 3. 修复图片文件名简化碰撞问题

建议在 SKILL.md 的步骤 3 "Download images + fix paths" 中补充：当多个图片 URL 映射到相同的简化本地文件名时，必须添加序号后缀避免冲突（如 `4584x2580-1.png`, `4584x2580-2.png`）。当前脚本将 4 张不同的 4584x2580 图片映射为同一引用名，导致引用与文件不匹配。

### 4. 添加 Windows 文件名安全字符替换规则

建议在 SKILL.md 的常见错误部分增加：脚本在保存图片文件前必须对文件名做安全字符替换——Windows 不允许文件名包含 `:`, `<`, `>`, `|`, `?`, `*` 等字符。byteimg CDN URL 中的 `~tplv-xxx:0:0:0:0:5o6Y6YeR...:q75.awebp` 含有 `:` 字符，导致 OSError 崩溃。涉及步骤 3 的 download_image() 函数。

### 5. 确保 WordPress 图片表格转换为独立图片引用
